# Update CV

Keep `profile/MASTER_CV.md` current as the user's work evolves. The master CV is
the single source every tailored resume is cut from; keep it complete and honest.

## When to use

- The user ships a new project, role, or notable accomplishment.
- A project's scope grew (rename, new capability, bigger impact).
- The user points you at new repos to fold in.

## Process

1. Read `profile/MASTER_CV.md` and `profile/PROFILE.md`.
2. For new/changed projects, read the source / README / git history (use `gh` for
   GitHub) so bullets are specific and accurate, not generic.
3. Add or revise entries in reverse-chronological order, quantified where possible
   (scale, time saved, users, revenue, adoption).
4. Respect honesty rules in PROFILE.md: real years, no claimed ownership the user
   didn't have, precise tool framing.
5. If skills changed materially, update `profile/ranking_profile.json` too so job
   ranking stays aligned.
6. Match the existing CV formatting; don't introduce LLM-isms or em-dash separators.

## After updating

Mention which tailored resumes are now stale and worth regenerating from the
refreshed master.
